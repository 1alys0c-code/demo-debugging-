Lab Debugging JavaScript

Memahami Tipe Data & Validasi Input

Masalah Utama

Error: Input Bukan Angka

Kesalahan logika paling umum saat menerima input dari user adalah mencampuradukkan Teks (String) dengan Angka (Number).

Kode Salah 

let angka = prompt("Masukkan angka:");
let hasil = angka + 10;
alert("Hasil: " + hasil);

Jika user mengetik "5", hasilnya menjadi "510" karena JavaScript menganggapnya sebagai penggabungan teks.

Kode Benar 

let input = prompt("Masukkan angka:");
let angka = Number(input);

if (isNaN(angka)) {
  alert("Error: Harus angka!");
} else {
  alert("Hasil: " + (angka + 10));
}

💡 Penjelasan Teknis

1

Fungsi Number() memaksa tipe data teks dari prompt menjadi angka murni.

2

Fungsi isNaN() singkatan dari is Not a Number. Ini mengecek apakah hasil konversi gagal.

3

Struktur if-else memberikan perlindungan agar program tidak error saat menerima input yang aneh.

Klik tombol di atas untuk melihat bagaimana validasi bekerja secara nyata.
